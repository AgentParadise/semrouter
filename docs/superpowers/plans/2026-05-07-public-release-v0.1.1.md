# Public Release v0.1.1 (Layer A — Polish + Dependency Slimming) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Take the existing internal-grade `semrouter` v0.1.0 and turn it into a polished, publishable, GitHub-hosted Rust crate named `semrouterr` on crates.io with a **minimal default dependency footprint**. Two thrusts: (1) public-release polish — LICENSE, CHANGELOG, README rewrite, CONTRIBUTING, examples, doc comments, GitHub Actions CI, typed errors, crates.io publish; (2) aggressive dependency slimming — drop `HttpEmbedder` + `reqwest` + `tokio` + `anyhow` + `chrono` entirely; feature-gate `fastembed` and `clap`. Default lib-only consumers compile against ~12 crates instead of 254.

**Architecture:** Rename `semrouter → semrouterr`. Migrate from Gitea to GitHub @ AgentParadise (history squashed by default). Strip the HTTP embedder along with its async dep stack; tiny Rust packages with zero unnecessary deps is the brand. Keep the public `EmbeddingProvider` trait so anyone who wants HTTP rolls their own with their HTTP client of choice (one trait method, ~30 lines). Feature-gate `fastembed` (default-on for batteries-included consumers) and `clap` (`cli` feature, only the binary needs it). Add LICENSE + CHANGELOG + CONTRIBUTING + examples, doc-comment every public item, replace `String` errors in `EvalSuite` with a `thiserror` enum, wire GitHub Actions CI with a `--no-default-features` matrix to keep the lean build healthy, then publish v0.1.1 to crates.io.

**Tech Stack:** Rust 2021. Mandatory deps after slimming: `serde`, `serde_json`, `toml`, `thiserror`. Optional: `fastembed` (`fastembed` feature, default-on), `clap` (`cli` feature, default-on for the binary). Dropped entirely: `reqwest`, `tokio`, `anyhow`, `chrono`.

```rust
//! Timestamp helpers built on `std::time::SystemTime`.
//!
//! Replaces our prior `chrono` dependency. Two formats:
//! - `iso8601_now()` → "2026-05-07T17:42:05Z"
//! - `compact_now()` → "20260507_174205" (filename-safe)

use std::time::{SystemTime, UNIX_EPOCH};

/// Current UTC time as RFC 3339 / ISO 8601 ("YYYY-MM-DDTHH:MM:SSZ").
pub fn iso8601_now() -> String {
    let secs = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .map(|d| d.as_secs())
        .unwrap_or(0);
    format_iso8601(secs)
}

/// Current UTC time as a compact filename-safe string ("YYYYMMDD_HHMMSS").
pub fn compact_now() -> String {
    let secs = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .map(|d| d.as_secs())
        .unwrap_or(0);
    format_compact(secs)
}

/// Civil date/time from unix seconds. Algorithm: Howard Hinnant's date.h
/// "civil_from_days" — exact, no allocation, no leap-second weirdness.
fn civil(secs: u64) -> (i32, u32, u32, u32, u32, u32) {
    let days = (secs / 86_400) as i64;
    let secs_of_day = (secs % 86_400) as u32;
    let hour = secs_of_day / 3600;
    let minute = (secs_of_day % 3600) / 60;
    let second = secs_of_day % 60;

    // Hinnant's algorithm; year is in the proleptic Gregorian calendar.
    let z = days + 719_468;
    let era = if z >= 0 { z / 146_097 } else { (z - 146_096) / 146_097 };
    let doe = (z - era * 146_097) as u64;
    let yoe = (doe - doe / 1460 + doe / 36_524 - doe / 146_096) / 365;
    let y = yoe as i64 + era * 400;
    let doy = doe - (365 * yoe + yoe / 4 - yoe / 100);
    let mp = (5 * doy + 2) / 153;
    let d = (doy - (153 * mp + 2) / 5 + 1) as u32;
    let m = (if mp < 10 { mp + 3 } else { mp - 9 }) as u32;
    let y = (y + if m <= 2 { 1 } else { 0 }) as i32;

    (y, m, d, hour, minute, second)
}

fn format_iso8601(secs: u64) -> String {
    let (y, mo, d, h, mi, s) = civil(secs);
    format!("{:04}-{:02}-{:02}T{:02}:{:02}:{:02}Z", y, mo, d, h, mi, s)
}

fn format_compact(secs: u64) -> String {
    let (y, mo, d, h, mi, s) = civil(secs);
    format!("{:04}{:02}{:02}_{:02}{:02}{:02}", y, mo, d, h, mi, s)
}

#[cfg(test)]
mod tests {

    #[test]

        // (verify with: date -u -j -f "%Y-%m-%dT%H:%M:%SZ" "2026-05-07T17:42:05Z" "+%s")
        assert_eq!(format_iso8601(1_778_262_125), "2026-05-07T17:42:05Z");
    }

    #[test]
    fn compact_format_for_2026() {
        assert_eq!(format_compact(1_778_262_125), "20260507_174205");
    }

        assert!(s.ends_with('Z'));

        assert_eq!(&s[7..8], "-");

        assert_eq!(&s[13..14], ":");
        assert_eq!(&s[16..17], ":");
    }
}

# semrouterr

[![CI](https://github.com/AgentParadise/semrouterr/actions/workflows/ci.yml/badge.svg)](https://github.com/AgentParadise/semrouterr/actions/workflows/ci.yml)
[![crates.io](https://img.shields.io/crates/v/semrouterr.svg)](https://crates.io/crates/semrouterr)
[![docs.rs](https://docs.rs/semrouterr/badge.svg)](https://docs.rs/semrouterr)

A lightweight, file-based semantic router for agent / model / workflow dispatch. Routes input text to a labeled route by comparing embeddings against a curated set of examples. **Zero default dependencies beyond `serde`, `serde_json`, `toml`, and `thiserror`** — bundle a local embedder via the `fastembed` feature, or bring your own. No LLM in the hot path. Sub-millisecond routing.

```
input text  →  embed  →  cosine vs. examples  →  top-K per route  →  threshold + margin  →  decision
```

## Why

If you're building an AI agent, voice assistant, or workflow system, you need to dispatch user input to one of N specialized handlers. The naive options — keyword matching (brittle), LLM classifier (slow, expensive, cloud round-trip) — both have real costs. semrouterr splits the difference: a tiny local embedding model gives you semantic understanding, and a flat file of labeled examples gives you a router you can edit and version-control.

semrouterr is a **pure classifier**. Risk classification, confirmation prompts, and dispatch live in your application — they don't belong in the router. This separation keeps risk policies next to the code that actually runs the dangerous thing.

## Install

**Batteries-included (default — bundles fastembed):**

```toml
[dependencies]
semrouterr = "0.1"
```

**Lean (lib only — bring your own `EmbeddingProvider`):**

```toml
[dependencies]
semrouterr = { version = "0.1", default-features = false }
```

The lean profile compiles against ~12 crates. The default profile pulls in fastembed (~190 crates including ONNX runtime) for batteries-included local embeddings.

## Bring your own embedder

The `EmbeddingProvider` trait is one method:

```rust
pub trait EmbeddingProvider: Send + Sync {
    fn embed(&self, text: &str) -> Result<Vec<f32>, RouterError>;
}
```

A custom HTTP-backed provider with `ureq` (5 transitive crates):

```rust
use semrouterr::embedding::EmbeddingProvider;
use semrouterr::error::RouterError;

struct OpenAIEmbedder { api_key: String }

impl EmbeddingProvider for OpenAIEmbedder {
    fn embed(&self, text: &str) -> Result<Vec<f32>, RouterError> {
        let resp: serde_json::Value = ureq::post("https://api.openai.com/v1/embeddings")
            .set("Authorization", &format!("Bearer {}", self.api_key))
            .send_json(serde_json::json!({
                "input": text,
                "model": "text-embedding-3-small"
            }))
            .map_err(|e| RouterError::Embedding(e.to_string()))?
            .into_json()
            .map_err(|e| RouterError::Embedding(e.to_string()))?;

        Ok(resp["data"][0]["embedding"]
            .as_array().unwrap()
            .iter().map(|v| v.as_f64().unwrap() as f32).collect())
    }
}
```

That's the full HTTP-embedder surface. semrouterr doesn't ship one because every consumer wants different things from their HTTP client (retry, batching, observability) — pick yours.
